How it works

Three components. One secure loop.

No polling, no SSH tunnels. An outbound WebSocket from each host carries everything.

01

Hook shim intercepts each tool call

Claude Code, Codex CLI, or OpenCode calls the agentcockpit hook shim before every tool use. The shim connects to the local daemon via Unix socket and blocks, waiting for a decision. Fails open if the daemon isn't running.

02

Relay delivers it to your browser

The daemon forwards the approval request over an outbound WebSocket to the relay server. The relay persists it and fans it out live to every browser tab you have open — no polling required.

03

You decide. The agent continues.

Allow or deny in one click. The decision flows back through the relay to the daemon, which unblocks the shim in milliseconds. The agent continues — or stops — based on your call.

Security

The relay routes. It cannot read.

Tool payloads and terminal output are end-to-end encrypted between your agent daemon and your browser. The relay server sees only routing metadata — never your actual content.

✓ Private key never leaves your browser — derived from your password with PBKDF2
✓ Auth tokens stored as SHA-256 hashes — raw tokens never persisted
✓ WebSocket connections use TLS in transit
✓ Open source — audit the relay and the hook shim yourself
